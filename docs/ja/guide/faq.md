# よくある質問

## `DicomFile` と `DataSet` のどちらから始めるべきですか?

ファイルやセッションの状態、ピクセル デコード、シリアライズを扱うなら `DicomFile` から始めてください。メタデータの読み書きに集中したいなら `DataSet` から始めます。

## `ds[...]` ではなく `get_value()` を使うのはどのような場合ですか?

一度だけ型付きで値を読みたいときは `get_value()` を使います。`tag`、`vr`、`length`、または生の要素メタデータも必要なら `ds[...]` を使います。

## 欠落した要素にアクセスしても常に例外が送出されないのはなぜですか?

DicomSDL は、多くの場面で `DataElement` へのアクセスがすぐに例外を送出しないよう設計されているため、真偽値判定で存在確認したり、検索を安全にチェーンしたりできます。

## 長さゼロの値が欠損値と異なって見えるのはなぜですか?

長さがゼロの場合は「存在するが空である」ことを意味し、欠落している場合は要素がまったく存在しないことを意味するためです。
